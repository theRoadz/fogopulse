# Story 2.11.4: Implement advance_epoch Instruction

Status: done

## Story

As a system,
I need the epoch to transition from Open to Frozen state when freeze_time is reached,
So that trading stops during the freeze window and settlement can proceed.

## Background

The original architecture (architecture.md, prd.md) describes `advance_epoch` as an atomic instruction that:
1. Settles the current epoch
2. Creates the next epoch

However, the implementation split this into separate instructions:
- `settle_epoch` (Story 3-1) - settles an epoch
- `create_epoch` (Story 1-6) - creates a new epoch

**What was missed:** The `Open → Frozen` state transition. The PRD says this is "auto by time" but there's no automatic on-chain mechanism - it needs an instruction.

**Root Cause:** Story 3-2 is titled "Implement confidence-aware refund logic" - NOT `advance_epoch`. The `advance_epoch` instruction was referenced in architecture comments but never added to any epic.

**Discovery:** While testing `settle_epoch` (Story 3.1), the instruction failed with:
```
ERROR: Epoch is in Open state, not Frozen.
Only epochs in Frozen state can be settled.
```

This is because `settle_epoch` requires epochs in `Frozen` state, but no instruction transitions epochs from `Open` → `Frozen`.

## Acceptance Criteria

1. **AC1**: `advance_epoch` instruction transitions epoch from `Open` to `Frozen` state
2. **AC2**: Instruction is permissionless (any wallet can call)
3. **AC3**: Instruction validates `clock.unix_timestamp >= epoch.freeze_time`
4. **AC4**: Instruction updates `pool.active_epoch_state` cache to `2` (Frozen)
5. **AC5**: Instruction emits `EpochFrozen` event
6. **AC6**: Instruction validates protocol/pool not frozen
7. **AC7**: Instruction rejects if epoch not in `Open` state
8. **AC8**: `advance-epoch.ts` script works with `--pool` CLI arg
9. **AC9**: Integration test verifies state transition

## Dev Notes

### Instruction Signature

```rust
pub fn advance_epoch(ctx: Context<AdvanceEpoch>) -> Result<()>
```

**No arguments needed** - this is purely time-based, no oracle required.

### Account Layout

```rust
#[derive(Accounts)]
pub struct AdvanceEpoch<'info> {
    /// Anyone can call - permissionless for crank bots/keepers
    #[account(mut)]
    pub payer: Signer<'info>,

    /// GlobalConfig - for freeze checks
    #[account(
        seeds = [b"global_config"],
        bump = global_config.bump,
    )]
    pub global_config: Box<Account<'info, GlobalConfig>>,

    /// Pool - must have this epoch as active
    #[account(
        mut,
        seeds = [b"pool", pool.asset_mint.as_ref()],
        bump = pool.bump,
        constraint = pool.active_epoch == Some(epoch.key()) @ FogoPulseError::InvalidEpoch,
    )]
    pub pool: Box<Account<'info, Pool>>,

    /// Epoch - must be in Open state
    #[account(
        mut,
        seeds = [b"epoch", pool.key().as_ref(), &epoch.epoch_id.to_le_bytes()],
        bump = epoch.bump,
        constraint = epoch.pool == pool.key() @ FogoPulseError::InvalidEpoch,
        constraint = epoch.state == EpochState::Open @ FogoPulseError::InvalidEpochState,
    )]
    pub epoch: Account<'info, Epoch>,

    /// Clock sysvar for timestamp
    pub clock: Sysvar<'info, Clock>,
}
```

### Handler Logic

```rust
pub fn handler(ctx: Context<AdvanceEpoch>) -> Result<()> {
    let config = &ctx.accounts.global_config;
    let pool = &mut ctx.accounts.pool;
    let epoch = &mut ctx.accounts.epoch;
    let clock = &ctx.accounts.clock;

    // Freeze checks
    require!(!config.frozen, FogoPulseError::ProtocolFrozen);
    require!(!pool.is_frozen, FogoPulseError::PoolFrozen);

    // Timing validation
    require!(
        clock.unix_timestamp >= epoch.freeze_time,
        FogoPulseError::EpochNotFrozen  // New error needed
    );

    // State transition
    epoch.state = EpochState::Frozen;
    pool.active_epoch_state = EpochState::Frozen.as_pool_cache_u8();

    // Emit event
    emit!(EpochFrozen {
        epoch: epoch.key(),
        pool: pool.key(),
        epoch_id: epoch.epoch_id,
        freeze_time: epoch.freeze_time,
    });

    Ok(())
}
```
