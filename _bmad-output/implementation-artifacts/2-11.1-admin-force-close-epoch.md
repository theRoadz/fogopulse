# Story 2.11.1: Admin Force-Close Epoch Instruction

Status: done

## Story

As an admin,
I want to force-close stuck epochs,
so that new epochs can be created when settlement is not yet implemented.

## Background

Epic 3 (Settlement & Payouts) is in backlog. Without `settle_epoch`, epochs stay active forever after `end_time` passes. This blocks new epoch creation since `pool.active_epoch` is never cleared.

This is a temporary admin utility until full settlement is implemented.

## Acceptance Criteria

1. **Admin-only Access**: Only the protocol admin (from GlobalConfig) can call this instruction
2. **Clears Pool State**: Sets `pool.active_epoch = None` and `pool.active_epoch_state = 0`
3. **Sets Epoch Refunded**: Sets `epoch.state = EpochState::Refunded`
4. **Emits Event**: Emits `EpochForceClosed` event for tracking
5. **No Payout Logic**: Does not handle position payouts (placeholder for Epic 3)
6. **Testnet Script**: TypeScript script to invoke from command line

## Dev Notes

### Accounts Structure

```rust
#[derive(Accounts)]
pub struct AdminForceCloseEpoch<'info> {
    #[account(mut)]
    pub admin: Signer<'info>,

    #[account(
        seeds = [b"global_config"],
        bump = global_config.bump,
        has_one = admin @ FogoPulseError::Unauthorized,
    )]
    pub global_config: Account<'info, GlobalConfig>,

    #[account(
        mut,
        seeds = [b"pool", pool.asset_mint.as_ref()],
        bump = pool.bump,
        constraint = pool.active_epoch == Some(epoch.key()) @ FogoPulseError::InvalidEpoch,
    )]
    pub pool: Account<'info, Pool>,

    #[account(
        mut,
        seeds = [b"epoch", pool.key().as_ref(), &epoch.epoch_id.to_le_bytes()],
        bump = epoch.bump,
    )]
    pub epoch: Account<'info, Epoch>,
}
```

### Handler Logic

```rust
pub fn handler(ctx: Context<AdminForceCloseEpoch>) -> Result<()> {
    let pool = &mut ctx.accounts.pool;
    let epoch = &mut ctx.accounts.epoch;

    // Mark epoch as refunded
    epoch.state = EpochState::Refunded;

    // Clear pool active epoch
    pool.active_epoch = None;
    pool.active_epoch_state = 0;

    emit!(EpochForceClosed {
        epoch: epoch.key(),
        pool: pool.key(),
        epoch_id: epoch.epoch_id,
        admin: ctx.accounts.admin.key(),
    });

    Ok(())
}
```

### Related: admin_close_epoch (added in Story 5-7.1)

A sibling instruction `admin_close_epoch` was added later to **delete** orphaned epoch accounts entirely (not just change state). This is needed after pool reinitialization: when pools are closed and recreated, `next_epoch_id` resets to 0, but old epoch PDAs still exist on-chain. `admin_close_epoch` closes the account and returns rent to admin. See `anchor/scripts/close-orphaned-epochs.ts` for the cleanup script.

- `admin_force_close_epoch` = changes epoch state to Refunded, clears pool.active_epoch (for stuck active epochs)
- `admin_close_epoch` = deletes the epoch account entirely, returns rent (for orphaned epochs after pool reinit)
