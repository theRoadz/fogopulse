# Story 2.11.2: Fix Pyth Lazer API Route 500 Error

Status: done

## Story

As a user,
I want the "Create New Epoch" button to work correctly,
so that I can create new trading epochs without encountering server errors.

## Problem

When clicking "Create New Epoch" button on `/trade/btc`, the application returns:
```
POST http://localhost:3000/api/pyth-price 500 (Internal Server Error)
```

## Root Cause Analysis

The API route at `web/src/app/api/pyth-price/route.ts:58` has two issues:

1. **Wrong WebSocket URL**: Uses `wss://pyth-lazer.dourolabs.app/v1/stream` but Pyth Lazer requires numbered endpoints:
   - `wss://pyth-lazer-0.dourolabs.app/v1/stream`
   - `wss://pyth-lazer-1.dourolabs.app/v1/stream`
   - `wss://pyth-lazer-2.dourolabs.app/v1/stream`

2. **Wrong auth method**: Passes token as URL query param (`?token=...`) but Pyth SDK expects token in config object
