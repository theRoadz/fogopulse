# Story 2.12.1: Fix Per-Side Cap Semantics

Status: done

## Story

As a **trader**,
I want to **place trades on pools with 50/50 seed liquidity**,
so that **trading is not blocked when the pool is perfectly balanced**.

## Bug Description

The per-side cap check is mathematically broken for 50/50 seeded pools.

With pool seeded at 10K up / 10K down (50/50 split):
- `pool_total = 20K`
- `side_cap_bps = 3000` (30%)
- `max_side = 20K * 3000 / 10000 = 6K`
- Current UP side = **10K**
- `10K > 6K` → **FAIL (ExceedsSideCap / 0x1788)**

**Both sides are at 50% which exceeds the 30% cap. Trading is impossible.**

## Root Cause

**Cap check:** `anchor/programs/fogopulse/src/utils/caps.rs:52-66`
```rust
let max_side = pool_total * cap_bps / 10000;
require!(side_total <= max_side, FogoPulseError::ExceedsSideCap);
```

**Seed liquidity:** `anchor/programs/fogopulse/src/instructions/admin_seed_liquidity.rs:140-154`
```rust
// 5. Split 50/50 between YES and NO reserves
let half_amount = amount / 2;
pool.yes_reserves += half_amount + (amount % 2);
pool.no_reserves += half_amount;
```

The seed instruction creates 50/50 split, but cap check requires each side ≤30%.

## Acceptance Criteria

1. **Given** a pool seeded with 50/50 liquidity, **When** a user places a trade, **Then** the trade succeeds (deviation is 0%)
2. **Given** a pool with 65/35 split, **When** a user places a trade on the 65% side, **Then** the trade succeeds (within 30% deviation)

4. **Given** any pool state, **When** cap is checked, **Then** the cap protects against IMBALANCE, not absolute percentage

## Dev Notes

### New Cap Logic

```rust
pub fn check_side_cap(side_total: u64, pool_total: u64, cap_bps: u16) -> Result<()> {
    if pool_total == 0 {
        return Ok(());
    }

    // NEW: Calculate balanced side (50% of pool)
    let balanced_side = pool_total / 2;

    // NEW: Cap is max DEVIATION from balanced, not max absolute percentage
    // With cap_bps=3000 (30%), deviation allowed is 30% of balanced side
    // e.g., balanced=10K, deviation=3K, max_allowed=13K (65% of total)
    let max_deviation = balanced_side
        .checked_mul(cap_bps as u64)
        .ok_or(FogoPulseError::Overflow)?
        .checked_div(10000)
        .ok_or(FogoPulseError::Overflow)?;

    let max_allowed = balanced_side
        .checked_add(max_deviation)
        .ok_or(FogoPulseError::Overflow)?;

    require!(side_total <= max_allowed, FogoPulseError::ExceedsSideCap);
    Ok(())
}
```

```rust
#[test]
fn test_side_cap_balanced_pool() {
    // 50/50 pool should always pass (deviation = 0)

#[test]
fn test_side_cap_within_deviation() {
    // 650 on one side in 1000 total = 15% over balanced
    // With 30% cap, deviation up to 150 (65% of pool) allowed
    let result = check_side_cap(650_000_000, 1_000_000_000, 3000);
    assert!(result.is_ok());
}

#[test]
fn test_side_cap_exceeds_deviation() {
    // 700 on one side in 1000 total = 20% over balanced
    // With 30% cap, max is 650 (65% of pool)
    let result = check_side_cap(700_000_000, 1_000_000_000, 3000);
    assert!(result.is_err());
}
```
