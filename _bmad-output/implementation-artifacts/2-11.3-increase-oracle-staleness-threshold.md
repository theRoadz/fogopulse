# Story 2.11.3: Increase Oracle Staleness Threshold

Status: done

## Story

As a user,
I want the epoch creation to succeed without timing out,
so that I can create epochs reliably from the browser UI.

## Problem

After fixing the Pyth Lazer API route (Story 2-11-2), epoch creation still fails with:
```
OracleDataStale - publish time exceeds staleness threshold
Error code: 0x1780 (6016)
```

The on-chain `oracle_staleness_threshold_start` is set to **3 seconds**, which is too tight for browser-based transaction flow:

| Step | Typical Time |
|------|-------------|
| Pyth publishes price | 0ms |
| Server fetches via WebSocket | ~200-500ms |
| Browser builds transaction | ~100ms |
| User clicks "Approve" in wallet | 1-5+ seconds |
| Transaction reaches FOGO blockchain | ~500ms-2s |
| **Total** | **2-8 seconds** |

With a 3-second threshold, the transaction fails if the user takes more than ~2 seconds to approve.

## Root Cause

The program was initialized with `oracle_staleness_threshold_start = 3` seconds. This was appropriate for server-side scripts where signing is instant, but not for browser UX where users manually approve transactions.

## Solution

Add an `update_config` admin instruction to modify the `oracle_staleness_threshold_start` from 3 to 10 seconds. This provides comfortable margin for human interaction while still ensuring fresh prices (Pyth publishes every 200ms).

## Dev Notes

### UpdateConfigParams Design

Use optional fields so admin can update only specific values:

```rust
#[derive(AnchorSerialize, AnchorDeserialize, Clone, Default)]
pub struct UpdateConfigParams {
    pub treasury: Option<Pubkey>,
    pub insurance: Option<Pubkey>,
    pub trading_fee_bps: Option<u16>,
    pub lp_fee_share_bps: Option<u16>,
    pub treasury_fee_share_bps: Option<u16>,
    pub insurance_fee_share_bps: Option<u16>,
    pub per_wallet_cap_bps: Option<u16>,
    pub per_side_cap_bps: Option<u16>,
    pub oracle_confidence_threshold_start_bps: Option<u16>,
    pub oracle_confidence_threshold_settle_bps: Option<u16>,
    pub oracle_staleness_threshold_start: Option<i64>,
    pub oracle_staleness_threshold_settle: Option<i64>,
    pub epoch_duration_seconds: Option<i64>,
    pub freeze_window_seconds: Option<i64>,
    pub allow_hedging: Option<bool>,
}
```

### Context Structure

```rust
#[derive(Accounts)]
pub struct UpdateConfig<'info> {
    #[account(
        mut,
        seeds = [b"global_config"],
        bump = global_config.bump,
        has_one = admin @ FogoPulseError::Unauthorized,
    )]
    pub global_config: Account<'info, GlobalConfig>,

    pub admin: Signer<'info>,
}
```

### Handler Logic

```rust
pub fn handler(ctx: Context<UpdateConfig>, params: UpdateConfigParams) -> Result<()> {
    let config = &mut ctx.accounts.global_config;

    if let Some(v) = params.oracle_staleness_threshold_start {
        config.oracle_staleness_threshold_start = v;
    }
    // ... similar for other fields

    emit!(ConfigUpdated { ... });

    Ok(())
}
```
