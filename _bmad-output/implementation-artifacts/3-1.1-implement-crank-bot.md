## Story

As an operator,
I want a crank bot that automatically manages epoch lifecycle,
So that epochs are created, advanced to frozen state, and settled without manual intervention.

## Context

This story creates a **standalone crank bot** that runs independently of the main repository. The bot continuously monitors the BTC pool and executes the appropriate epoch lifecycle actions based on pool state and timing.

**Current State:**
- `create_epoch` instruction exists (Story 2.11) - creates new epoch with Pyth oracle
- `advance_epoch` instruction exists (Story 2.11.4) - transitions Open → Frozen at freeze_time
- `settle_epoch` instruction exists (Story 3.1) - settles with Pyth oracle at end_time
- Manual scripts exist: `create-test-epoch.ts`, `advance-epoch.ts`, `settle-epoch.ts`
- No automated bot to run the full lifecycle continuously

**What This Enables:**
- Continuous epoch lifecycle without manual intervention
- Reliable settlement ensuring users can claim payouts
- Permissionless operation (anyone can run the bot)
- Production readiness for mainnet deployment

**Deployment Targets:**
- Local PC (Windows) for testing
- Contabo VPS (Linux) for production

## Acceptance Criteria

1. **Given** a pool with no active epoch
   **When** the crank bot detects this state
   **Then** it creates a new epoch using `create_epoch` instruction with Pyth oracle data

2. **Given** an epoch in Open state that has passed its `freeze_time`
   **When** the crank bot detects this state
   **Then** it advances the epoch to Frozen state using `advance_epoch` instruction

3. **Given** an epoch in Frozen state that has passed its `end_time`
   **When** the crank bot detects this state
   **Then** it settles the epoch using `settle_epoch` instruction with Pyth oracle data

4. **Given** a settled epoch
   **When** the settlement completes and `pool.active_epoch` is cleared
   **Then** the bot creates a new epoch (full cycle completion)

5. **Given** a transient RPC or Pyth WebSocket error
   **When** the bot encounters the error
   **Then** it retries with exponential backoff (3 attempts max)
   **And** continues to the next poll cycle if all retries fail

6. **Given** a critical error (insufficient balance, missing wallet)
   **When** the bot encounters the error
   **Then** it logs the error and exits gracefully

7. **Given** the bot receives SIGINT or SIGTERM
   **When** shutdown signal is received
   **Then** the bot completes any in-progress transaction and exits gracefully

8. **Given** the bot is deployed on Contabo
   **When** it runs as a systemd service
   **Then** it auto-restarts on failure and logs to journald

### Pool State Mapping

```typescript
// Pool.active_epoch_state values
const POOL_STATE = {
  None: 0,      // No active epoch
  Open: 1,      // Trading allowed
  Frozen: 2,    // In freeze window
  Settling: 3,  // Settlement in progress
  Settled: 4,   // Outcome determined
  Refunded: 5,  // Oracle failed
}
```

### State Machine Logic

```typescript
enum Action {
  NONE,           // Nothing to do, wait
  CREATE_EPOCH,   // No active epoch
  ADVANCE_EPOCH,  // Open state + freeze_time passed
  SETTLE_EPOCH,   // Frozen state + end_time passed
}

function determineAction(poolState, epochData, currentTime): Action {
  if (poolState.activeEpoch === null) {
    return Action.CREATE_EPOCH
  }

  if (poolState.activeEpochState === POOL_STATE.Open) {
    if (currentTime >= Number(epochData.freezeTime)) {
      return Action.ADVANCE_EPOCH
    }
    return Action.NONE // Wait for freeze_time
  }

  if (poolState.activeEpochState === POOL_STATE.Frozen) {
    if (currentTime >= Number(epochData.endTime)) {
      return Action.SETTLE_EPOCH
    }
    return Action.NONE // Wait for end_time
  }

  return Action.NONE
}
```

### Ed25519 Instruction Pattern

For `create_epoch` and `settle_epoch`, Ed25519 verification instruction must be FIRST:

```typescript
// Ed25519 instruction references Pyth message in target instruction
const ed25519Ix = createEd25519Instruction(pythMessage, 1, 12)  // index=1, offset=12

// Transaction order: [Ed25519, TargetInstruction]
const instructions = [ed25519Ix, createOrSettleIx]
```

**Pyth Message Offset = 12:**
- 8 bytes: Anchor discriminator
- 4 bytes: Vec<u8> length prefix
- → Pyth message bytes start at offset 12

## Deployment Instructions

### Local (Windows)

```bash
cd crank-bot
npm install
cp .env.example .env
# Edit .env with PYTH_ACCESS_TOKEN

npx tsx crank-bot.ts
```

### Contabo (Linux)

```bash
# SSH to server
ssh user@contabo-ip

# Setup
mkdir ~/fogopulse-crank
scp crank-bot.ts package.json .env user@contabo-ip:~/fogopulse-crank/
cd ~/fogopulse-crank
npm install

# Copy wallet
mkdir -p ~/.config/solana
scp ~/.config/solana/fogo-testnet.json user@contabo-ip:~/.config/solana/

# Install service
sudo cp fogopulse-crank.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable fogopulse-crank
sudo systemctl start fogopulse-crank

# Monitor
sudo journalctl -u fogopulse-crank -f
```
